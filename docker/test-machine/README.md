# Test Machine Container

This Docker container simulates a physical machine for testing Orthos2's machine management features.

## Features

- **SSH Daemon**: Accepts passwordless root login via pre-configured SSH key
- **Hardware Tools**: dmidecode, hwinfo, lspci, lsscsi, lsusb for inventory scanning
- **Python Support**: Enables Ansible playbook execution
- **Minimal but Real Data**: Provides container's actual hardware view (minimal but realistic)

## Container Details

- **Hostname**: `testmachine.orthos2.test`
- **Base Image**: OpenSUSE Tumbleweed
- **SSH Port**: 22
- **Default User**: root (passwordless key authentication)
- **Network**: Connected to `orthos` Docker network

## SSH Authentication

The container uses automatically generated SSH keys located in `ssh-keys/`:
- Keys are generated by `docker/manage-secrets.py` (not committed to git)
- Private key is mounted into orthos2 and orthos2_taskmanager containers
- Public key is copied into container's `/root/.ssh/authorized_keys` during build

**IMPORTANT**: Run `python3 docker/manage-secrets.py` before building the container to generate SSH keys.

**WARNING**: These are test-only keys. Never use in production!

## Usage

### Start Container
```bash
docker compose up -d test_machine
```

### Test SSH Connection
```bash
docker compose exec orthos2 ssh -i /var/lib/orthos2/.ssh/orthos2-test-machine -o StrictHostKeyChecking=no root@testmachine.orthos2.test hostname
```

### Check Hardware Tools
```bash
docker compose exec orthos2 ssh -i /var/lib/orthos2/.ssh/orthos2-test-machine -o StrictHostKeyChecking=no root@testmachine.orthos2.test "dmidecode --version && hwinfo --version"
```

## Integration with Orthos2

The test machine is automatically registered in Orthos2 database via fixtures:
- `orthos2/data/fixtures/tests/test_domain_orthos2test.json` - Domain configuration
- `orthos2/data/fixtures/tests/test_machine_docker.json` - Machine and network interface

FQDN: `testmachine.orthos2.test`

## Testing Features

This container allows testing:
- SSH connectivity and authentication
- Hardware inventory collection via Ansible
- Remote command execution (shutdown, reboot, scripts)
- Machine status checks (ping, SSH port, login)
- File transfers (SFTP, SCP)
- Ansible fact collection and parsing

## Installed Tools

- openssh-server - SSH daemon
- iproute2 - Network configuration (ip command)
- systemd - Init system (for shutdown/reboot)
- dmidecode - DMI/SMBIOS data
- hwinfo - Hardware information
- pciutils - PCI device listing (lspci)
- usbutils - USB device listing (lsusb)
- lsscsi - SCSI device listing
- util-linux - System utilities
- python3 - Python interpreter for Ansible
